# Исправление ошибки git pull на сервере

## Проблема
При выполнении `git pull` на сервере возникает ошибка:
```
error: Your local changes to the following files would be overwritten by merge:
src/config.py
Please commit your changes or stash them before you merge.
```

## Решение

На сервере выполните одну из команд:

### Вариант 1: Сохранить локальные изменения и применить их после pull (рекомендуется)
```bash
# Сохраняем локальные изменения
git stash

# Делаем pull
git pull

# Применяем сохраненные изменения обратно
git stash pop

# Если есть конфликты, разрешите их вручную
```

### Вариант 2: Откатить локальные изменения (если они не нужны)
```bash
# Откатываем изменения в config.py
git restore src/config.py

# Делаем pull
git pull
```

### Вариант 3: Закоммитить изменения (если они нужны)
```bash
# Добавляем изменения
git add src/config.py

# Коммитим
git commit -m "Update config.py: улучшено чтение .env и добавлено логирование"

# Делаем pull (может потребоваться merge)
git pull

# Если есть конфликты, разрешите их и сделайте:
git add src/config.py
git commit -m "Merge: разрешены конфликты в config.py"
```

## Рекомендация

Используйте **Вариант 1** (stash), так как он сохранит ваши локальные изменения на сервере (например, настройки из .env), но применит обновления из GitHub.

После применения изменений проверьте, что настройки в `.env` на сервере правильные.
